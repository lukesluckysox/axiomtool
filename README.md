# Axiom

**The fourth and final tool in the Axiom OS ecosystem.**

> Liminal questions belief · Parallax reveals pattern · Praxis tests hypothesis · **Axiom distills truth**

---

## What is Axiom?

Axiom is the synthesis layer of the operating system. It is the place where insight, pattern recognition, and lived experimentation are consolidated into durable principles.

Where Liminal destabilizes assumptions, Parallax identifies recurring patterns, and Praxis generates evidence through lived experiments, Axiom gathers those outputs and turns them into a coherent structure of truth.

Axiom is **not** a notes dashboard, a second brain, a PKM tool, or a quote collector. It is a system for synthesizing what has actually been learned.

## Core Synthesis Model

Every synthesis cycle follows this structure:

**Signal → Convergence → Interpretation → Principle → Revision**

## Core Objects

- **Truth Claims** — statements that appear increasingly reliable across evidence
- **Working Principles** — actionable doctrines the user can live by
- **Core Tensions** — unresolved polarities that continue to organize the user's life
- **Contradictions** — areas where evidence is mixed or the self remains divided
- **Revisions** — records of how the user's worldview has changed over time
- **Constitution** — the current operating structure of the self

## Stack

- **Frontend**: React + Vite + Tailwind CSS + shadcn/ui + wouter
- **Backend**: Express + better-sqlite3 + Drizzle ORM
- **Fonts**: General Sans (Fontshare) + Cormorant Garamond + IBM Plex Mono

## Development

```bash
npm install
npm run dev
```

The app runs on `http://localhost:5000`.

## Philosophy

The goal of Axiom is not inspiration, optimization, or self-branding.

Its goal is to help the user build a personal philosophy rigorous enough to live by.

Truth in Axiom is never arbitrary and never mystical. It is synthesized from evidence.
